Configuration

Centralize your application logic. ArikaJS provides a robust configuration system that seamlessly merges your secret environment variables with your application's structural settings.

The Config Architecture

All of your application's structural settings reside within the config directory. By default, ArikaJS provides specialized files for every core system (Database, Auth, Cache, etc.). These files are automatically loaded into the framework's Service Container during the bootstrap process.

Environment Management

Sensitive data like API keys and database passwords should never be hardcoded. Use the .env file provided at the root of your project. ArikaJS uses a sophisticated loader that automatically converts common string values into their native types:

.env
APP_DEBUG=true  # Converted to Boolean
APP_PORT=3000   # Converted to Number
DB_HOST=127.0.0.1

Creating Custom Config Files

You are encouraged to create your own configuration files within the config directory for third-party services. For instance, if you're using Stripe, create a config/stripe.ts:

config/stripe.ts
export default {
    api_key: env('STRIPE_KEY'),
    webhook_secret: env('STRIPE_WEBHOOK')
};

Configuration Helpers

ArikaJS provides two primary helpers for interacting with your settings:

  • config(): Used to retrieve nested values using dot notation (e.g., config('database.connections.mysql.host')).
  • env(): Used to retrieve raw values from your .env file with an optional fallback.
