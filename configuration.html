Configuration

Centralize your application's settings. ArikaJS provides a robust configuration engine that merges environment variables with static configuration files.

Introduction

All of the configuration for ArikaJS is stored in the config directory at the root of your project. This directory typically contains files for your database, sessions, mail, and other core services. Each file returns a configuration object that is automatically registered into the Service Container during bootstrap.

Environment Variables

Secure and flexible applications rely on environment variables. ArikaJS loads these from your .env file. You should never commit this file to version control.

.env
APP_NAME=ArikaJS
APP_ENV=local
APP_KEY=base64:7ca...
APP_DEBUG=true
APP_URL=http://localhost:3000

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306

Accessing Configuration

You may use the config() helper from anywhere in your application. It supports "dot" notation to drill down into nested configuration objects:

Example Usage
import { config } from 'arikajs';

// Retrieve app name
const name = config('app.name');

// Retrieve with default value fallback
const timezone = config('app.timezone', 'UTC');

The Application Key

The APP_KEY is critical for securing user sessions and encrypted data. If this key is missing or incorrect, ArikaJS will throw a fatal error during startup for security reasons. Generate a unique key using the CLI:

Terminal
arika key:generate
