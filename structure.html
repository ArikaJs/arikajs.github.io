Directory Structure

ArikaJS follows a strict architectural pattern ensure separation of concerns. Understanding the directory layout is key to building maintainable large-scale applications.

The App Layer

The app directory contains the core logic of your application. This is where you will spend 90% of your development time:

  • Http/Controllers: Responsible for processing incoming requests and returning responses.
  • Http/Middleware: Logic for filtering and transforming incoming requests (Auth, Logs).
  • Http/Requests: "FormRequest" classes that handle request validation and authorization.
  • Models: Data entities mapped to your database using the Arika ORM.
  • Providers: Service Providers that bind core services into the container.

Configuration Files

The config/ directory contains the DNA of your application's settings. Here is a breakdown of the default files:

  • app.ts: Name, Environment (local/prod), Key, and global URL.
  • auth.ts: Authentication guards (session, jwt) and model providers.
  • cache.ts: Store definitions (Redis, Memcached, File) for caching.
  • database.ts: Connection strings and pool settings for SQL engines.
  • filesystems.ts: Disk declarations for Local and S3 storage.
  • logging.ts: Channel management (Stack, Daily, Slack) for error reporting.
  • mail.ts: Transport logic for SMTP, Mailgun, and SparkPost.
  • queue.ts: Job worker settings for asynchronous processing.

Framework Directories

  • bootstrap/: Script that initializes the Kernel and sets up the container.
  • storage/: Generated assets. Contains logs, framework/sessions, and framework/views.
  • public/: The web root. Contains index.ts and built frontend assets.
