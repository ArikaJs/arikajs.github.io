Directory Structure

The default ArikaJS application structure is engineered to scale. It follows strict separation of concerns, ensuring your code remains testable and maintainable.

The Root Directories

Project Root
.
├── app/            # Application Source Code
├── bootstrap/      # Application Bootstrapping
├── config/         # Configuration Files
├── database/       # Migrations & Seeds
├── public/         # Entry point (index.ts)
├── resources/      # Views & Assets
├── routes/         # Web & API Routes
└── storage/        # Cache, Logs & Session data

The App Directory

The app directory contains the core logic of your application. Almost all of your business logic will live here:

  • Http/: Contains your Controllers, Middleware, and FormRequests. This is the entry point for your web logic.
  • Models/: Contains the Arika ORM model classes.
  • Providers/: Contains Service Providers which bootstrap the framework components.
  • Services/: A recommended place for complex business logic that doesn't belong in controllers.

Resources & Views

The resources directory contains your Ark Templates and raw frontend assets. When you build your application, these are processed and served from public/dist.
