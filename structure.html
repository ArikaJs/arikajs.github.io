Directory Structure

Master the ArikaJS application layout. Every folder in the skeleton has a specific architectural purpose designed to separate concerns and maximize development speed.

The Root Directory

The root directory contains the following core folders and files:

  • app/: The house for your application's domain logic.
  • bootstrap/: Logic for framework initialization and service registration.
  • config/: Centralized settings for all framework components.
  • database/: Migrations, data seeds, and model factories.
  • public/: The document root containing index.ts and public assets.
  • resources/: Raw assets (CSS/JS) and Ark templates.
  • routes/: Definitions for web and API endpoints.
  • storage/: Temporary files such as logs, session data, and cache.

The App Directory

The heart of your application. Almost every class you write will live in a sub-folder here:

Directory Overview
app/
├── Http/
│   ├── Controllers/    # Handles incoming HTTP requests
│   ├── Middleware/     # Filters & guards for requests
│   └── Requests/       # Custom form validation logic
├── Models/             # Database entities (Fluent ORM)
├── Providers/          # Service Providers (Bindings)
└── Services/           # Reusable business logic services

The Config Directory

ArikaJS provides individual configuration files for every core module. Each file allows you to switch drivers and set security parameters:

  • app.ts: Core settings like URL, timezone, and locale.
  • auth.ts: Configuration for guards (drivers) and user providers.
  • cache.ts: Config for Redis, File, or Memory caching.
  • database.ts: Connection settings for MySQL, PostgreSQL, etc.
  • filesystems.ts: Disk management for local or cloud storage (S3).
  • logging.ts: Error and activity logging channels.
  • mail.ts: SMTP or API settings for sending emails.
  • queue.ts: Configuration for background job processing.

The Routes Directory

By default, ArikaJS provides two route files to keep your endpoints organized:

  • web.ts: Routes for your browser-based interface (includes sessions & CSRF).
  • api.ts: Routes for your API (includes rate limiting and token auth).
