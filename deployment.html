Deployment

Ready to go live? ArikaJS follows standard Node.js deployment practices but provides specialized optimizations for high-performance production environments.

The Build Process

Before deploying to production, your TypeScript application must be compiled into JavaScript. This ensures maximum execution speed is achieved by the Node.js V8 engine:

Deployment
# Compile TypeScript to JavaScript
npm run build

# Prune development dependencies
npm install --production

Process Management

For production servers, we recommend using a process manager like PM2. PM2 will automatically restart your application if it crashes and allows you to perform zero-downtime reloads.

PM2 Ecosystem
pm2 start dist/index.js --name arikajs-app

Nginx Configuration

It is best practice to serve your ArikaJS application behind an Nginx reverse proxy. This allows Nginx to handle SSL termination, static file caching, and load balancing:

Nginx Config
server {
    listen 80;
    server_name example.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
