Routing

Expressive and powerful routing for your Node.js applications. ArikaJS routes provide a clean, fluent interface for mapping URIs to specific application behavior.

Basic Routing

The most fundamental ArikaJS routes accept a URI and a closure (or controller method). By default, web-specific routes are defined in routes/web.ts:

routes/web.ts
import { Route } from 'arikajs';

Route.get('/welcome', (req, res) => {
    return 'Welcome to ArikaJS!';
});

Route Parameters

Capture dynamic URI segments using the colon notation. These parameters are injected directly into the request object:

Route.get('/user/:id', (req) => {
    return 'User ID: ' + req.params.id;
});

Route Groups

Route groups allow you to share path prefixes or middleware across a large number of routes without needing to define those attributes on each individual route:

Route.prefix('admin').group(() => {
    Route.get('/dashboard', () => { /* /admin/dashboard */ });
    Route.get('/users', () => { /* /admin/users */ });
});

Middleware Assignment

You may assign middleware to a route or group to filter incoming requests before they reach your controller logic:

Route.get('/profile', () => {
    // ...
}).middleware('auth');
