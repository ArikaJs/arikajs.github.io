Basic Routing

The most basic ArikaJS routes accept a URI and a closure, providing a very simple and expressive method of defining routes and behavior without complicated routing configuration files.

Basic Routing

The default ArikaJS routes are defined in your routes/web.ts and routes/api.ts files. The web.ts file defines routes that are for your web interface. These routes are assigned the web middleware group, which provides features like session state and CSRF protection.

routes/web.ts
import { Route } from 'arikajs';

Route.get('/greeting', () => {
    return 'Hello World';
});
💡
Pro Tip: Most of your application's routes will be defined in the web.ts file, as this handles standard browser requests.

Available Router Methods

The router allows you to register routes that respond to any HTTP verb:

Route.get(uri, callback);
Route.post(uri, callback);
Route.put(uri, callback);
Route.patch(uri, callback);
Route.delete(uri, callback);
Route.options(uri, callback);

Redirect Routes

If you are defining a route that redirects to another URI, you may use the Route.redirect method. This method provides a convenient shortcut so that you do not have to define a full route or controller for a simple redirect:

Route.redirect('/here', '/there');
