Middleware

Middleware provide a convenient mechanism for inspecting and filtering HTTP requests entering your application.

Introduction

Middleware act as a series of "layers" that a request must pass through before it reaches your controller. For example, ArikaJS includes middleware that verifies whether the user of your application is authenticated. If the user is not authenticated, the middleware will redirect the user to the login screen. However, if the user is authenticated, the middleware will allow the request to proceed further into the application.

Defining Middleware

To create a new middleware, use the make:middleware CLI command:

arika make:middleware EnsureTokenIsValid

A middleware is a class that implements the Middleware contract. It contains a handle method which receives the request and a next callback:

export class EnsureTokenIsValid {
    async handle(req, next) {
        if (req.header('token') !== 'secret') {
            return res.redirect('login');
        }

        return await next();
    }
}

Registering Middleware

Global middleware are listed in your app/Http/Kernel.ts file. If you want a middleware to run during every HTTP request to your application, list the middleware class in the $middleware property of this file.
